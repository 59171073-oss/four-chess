# 儿童重力四子棋 Android APP Spec

## Why
重力四子棋（Connect-4）是一款经典策略小游戏，适合培养儿童的逻辑思维与空间推理能力。参考项目 `https://github.com/xmxoxo/connect4` 为 Python/Web 实现，无法在移动端便捷使用，且其 AI 算法固定、缺乏针对儿童的难度自适应与个性化辅导能力。需要构建一款面向 Android 系统、适配儿童体验的重力四子棋 APP，并嵌入基于大模型 API 的 AI 棋手，实现难度随儿童能力动态调整，并分析棋路提供针对性提升建议。

## What Changes
- 新建 Android 原生 APP 工程（Kotlin + Jetpack Compose），实现重力四子棋核心对弈逻辑（标准 7 列 × 6 行棋盘，重力下落，四子连珠胜负判定）
- 实现本地规则/启发式 AI 棋手，提供入门、简单、普通、困难等多个难度等级，支持离线对弈
- 嵌入基于大模型 API 的 AI 棋手，通过调用大模型接口生成落子决策与对话式互动
- 实现自适应难度系统：根据儿童近期对局表现（胜率、失误类型）动态调整 AI 难度
- 实现棋路分析模块：记录并分析儿童落子习惯与失误模式（未防守威胁、错失连子机会等）
- 提供针对性辅导功能：基于棋路分析给出提示、复盘建议与练习关卡
- 构建儿童友好的 UI/UX：大字号、明亮配色、可爱形象、动画与音效、鼓励性反馈
- 提供设置与数据持久化：玩家档案、对局历史、成绩统计、API Key/端点配置

## Impact
- Affected specs: 全新 APP，无既有 spec 受影响
- Affected code: 全新 Android 工程，核心对弈逻辑移植自参考项目 `https://github.com/xmxoxo/connect4` 的 `ailib.py`（Board 类、特征评估、2 步搜索）
- 外部依赖：大模型 API（用户需在设置中配置 API Key 与端点）
- 目标平台：Android 8.0 (API 26) 及以上
- 受众：6-12 岁儿童

## ADDED Requirements

### Requirement: 棋盘与对弈核心
系统 SHALL 提供标准重力四子棋棋盘（7 列 × 6 行），支持人机对弈的落子、换边、胜负与平局判定，规则移植自参考项目的 Board 实现。

#### Scenario: 棋子重力下落
- **WHEN** 玩家点击某一列
- **THEN** 棋子下落到该列最底部的空位；若该列已满则提示无效并禁止落子

#### Scenario: 四子连珠获胜
- **WHEN** 同色棋子在横、竖、左斜、右斜任一方向连成 4 个
- **THEN** 判定该方获胜，高亮连珠棋子并结束本局

#### Scenario: 棋盘满局平局
- **WHEN** 棋盘所有格子已填满且无四子连珠
- **THEN** 判定平局并结束本局

### Requirement: 本地 AI 难度等级
系统 SHALL 提供多档本地 AI 难度（入门、简单、普通、困难），基于参考项目的特征评估与搜索算法，通过调整搜索深度与策略随机性实现分级，全部支持离线对弈。

#### Scenario: 难度选择
- **WHEN** 用户开始新对局
- **THEN** 可选择难度等级，AI 的搜索深度与策略随等级变化（入门：随机+基础防守；困难：深层搜索+完整启发式）

#### Scenario: 入门难度行为
- **WHEN** 难度为入门
- **THEN** AI 以较高随机性落子，仅在临门威胁时防守，便于儿童获胜建立信心

### Requirement: 大模型 AI 棋手
系统 SHALL 嵌入一个通过调用大模型 API 进行落子决策的 AI 棋手，作为可选的"智能教练对手"。

#### Scenario: AI 落子决策
- **WHEN** 轮到大模型 AI 行棋且 API 已配置
- **THEN** APP 将当前棋盘状态、轮次、对局历史序列化为提示词，调用大模型 API，解析返回的列号执行落子

#### Scenario: API 不可用回退
- **WHEN** 大模型 API 调用失败、超时或未配置 API Key
- **THEN** 回退到本地 AI（按当前自适应难度）完成本手，保证对局不中断，并向用户/日志提示回退原因

#### Scenario: 对话式互动
- **WHEN** 对局结束或儿童请求帮助
- **THEN** 可调用大模型生成鼓励性、儿童易懂的复盘对话与建议

### Requirement: 自适应难度
系统 SHALL 根据儿童近期对局表现动态调整 AI 难度（适用于本地 AI 与大模型 AI 的强度参数）。

#### Scenario: 连胜升级
- **WHEN** 儿童在同一难度下连续获胜达到阈值（默认 2 局）
- **THEN** 自动提升一档难度并告知儿童

#### Scenario: 连败降级
- **WHEN** 儿童连续失利达到阈值（默认 3 局）
- **THEN** 自动降低一档难度并给予鼓励，避免挫败感

#### Scenario: 难度可手动覆盖
- **WHEN** 用户在设置中关闭"自适应难度"
- **THEN** 难度固定为用户所选等级，不自动调整

### Requirement: 棋路分析
系统 SHALL 记录每局落子序列并在对局结束后分析儿童的棋路特征与失误模式。

#### Scenario: 失误识别
- **WHEN** 一局对局结束
- **THEN** 系统识别并记录儿童的关键失误类型：未防守对手威胁、错失制胜机会、未控制中心列等

#### Scenario: 棋路画像
- **WHEN** 累计多局数据
- **THEN** 生成儿童棋路画像（常用列、偏好进攻/防守、常见失误分布），持久化存储

### Requirement: 针对性辅导
系统 SHALL 基于棋路分析为儿童提供针对性的提升帮助。

#### Scenario: 弱项提示
- **WHEN** 检测到儿童在某类失误上反复出现
- **THEN** 在对局结束后或首页推送针对性的简短提示卡片（如"记得看看对手有没有连成3个"）

#### Scenario: 复盘与练习
- **WHEN** 儿童点击"复盘"
- **THEN** 回放关键步并标注失误点；可选生成针对弱项的练习局面供儿童训练

### Requirement: 儿童友好界面
系统 SHALL 提供面向 6-12 岁儿童的友好 UI/UX。

#### Scenario: 视觉与交互
- **WHEN** 儿童使用 APP
- **THEN** 界面采用大字号、明亮配色、圆角与大点击区域，配可爱形象与落子动画、音效

#### Scenario: 鼓励性反馈
- **WHEN** 儿童落子或胜负判定
- **THEN** 给出正面、鼓励性反馈，避免负面措辞

### Requirement: 设置与数据持久化
系统 SHALL 提供设置页面与本地数据持久化。

#### Scenario: 玩家档案与历史
- **WHEN** 用户使用 APP
- **THEN** 本地持久化玩家档案、对局历史、成绩统计、棋路画像

#### Scenario: API 配置
- **WHEN** 用户进入设置
- **THEN** 可配置大模型 API 端点、API Key、模型名称，并测试连通性；密钥安全存储于本地
